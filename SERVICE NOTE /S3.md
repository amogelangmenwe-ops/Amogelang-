AWS Storage Services – Focus on S3
=


Amazon Simple Storage Service (S3) is a highly scalable, serverless object storage service designed to store and retrieve large amounts of data from anywhere on the web. Unlike traditional file systems, S3 abstracts away hardware management, allowing users to focus on storing data efficiently while paying only for what they use. It is ideal for use cases like hosting static websites, storing backups, or managing big data for analytics.

AWS provides several types of storage services to meet different needs. Object storage like S3 is designed for durability and scalability. Block storage such as EBS is optimized for persistent storage attached to EC2 instances, while file storage like EFS allows multiple EC2 instances to share a file system. For hybrid cloud environments, Storage Gateway extends on-premise storage to the cloud, and specialized services like Amazon FSx provide high-performance, feature-rich file systems for Windows or Linux workloads.

Within S3, storage classes allow you to optimize cost and performance depending on how frequently data is accessed. Standard S3 is designed for frequently accessed data, S3 Intelligent-Tiering automatically moves objects between tiers based on access patterns, and S3 Glacier and Glacier Deep Archive offer low-cost, long-term archival solutions for data that is rarely accessed. This flexibility makes S3 suitable for workloads ranging from active databases to long-term backups.

The AWS Snow Family complements S3 by enabling physical data migration to the cloud for large datasets. Devices like Snowball and Snowball Edge can transfer tens of terabytes, while Snowcone handles smaller-scale transfers, and Snowmobile, a massive shipping container on a truck, can move up to 100 petabytes. These devices are ideal for organizations that need to upload large datasets to S3 securely and efficiently without relying solely on network transfers.

| --------------------------------- | ---------------------- | -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **S3 (Simple Storage Service)**   | Object Storage         | Serverless storage for large files                 | Pay-as-you-go, no file system management needed, highly durable and scalable                                                                                               |
| **S3 Glacier**                    | Cold Storage           | Archiving and long-term backup                     | Low-cost, secure, durable, optimized for infrequent access                                                                                                                 |
| **EBS (Elastic Block Store)**     | Block Storage          | Persistent storage for EC2 instances               | Virtual hard drive, SSD options (General Purpose, IOPS), HDD options (Throughput HDD, Cold HDD)                                                                            |
| **EFS (Elastic File System)**     | File Storage           | Shared storage for multiple EC2 instances          | NFS-based, cloud-native, automatically scalable                                                                                                                            |
| **Storage Gateway**               | Hybrid Cloud Storage   | Extend on-premise storage to AWS                   | **File Gateway:** extend local file storage to S3<br>**Volume Gateway:** caches local drives to S3<br>**Tape Gateway:** virtual tapes for cost-effective long-term storage |
| **AWS Snow Family**               | Physical Data Transfer | Large-scale physical migration to AWS              | **Snowball/Snowball Edge:** 50–80 TB<br>**Snowcone:** 8 TB<br>**Snowmobile:** up to 100 PB per trailer                                                                     |
| **AWS Backup**                    | Backup Service         | Centralized backup for multiple AWS services       | Automates backups for EC2, EBS, RDS, DynamoDB, EFS, Storage Gateway; supports backup plans                                                                                 |
| **CloudEndure Disaster Recovery** | Disaster Recovery      | Continuous replication for fast recovery           | Replicates machines to a low-cost staging area in target AWS account and region                                                                                            |
| **Amazon FSx**                    | Managed File Systems   | High-performance file storage for Windows or Linux | **FSx for Windows:** SMB protocol, mounts to Windows servers<br>**FSx for Lustre:** Lustre file system, mounts to Linux servers                                            |
